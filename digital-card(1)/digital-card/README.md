作品名称
个人数字名片 · 袁军

作品简介
这是一个简洁、美观的个人数字卡片页。它模拟了现代软件开发者的个人简介页，包含了身份信息、兴趣标签、技能标签以及自我介绍。页面设计采用了毛玻璃效果和柔和阴影，旨在通过一张卡片的形式清晰展示一个人的核心信息。

页面主要内容说明
页面主要包含以下模块：

身份区：头像、姓名（含昵称）、学号、专业。

签名区：一句个性化的签名。

联系方式：占位联系方式。

标签区：3个兴趣标签（篮球、独立游戏、科幻）和3个技能标签（Java、Spring Boot、MySQL）。

自我介绍：超过100字的个人详细介绍。

外部链接：指向Gitee主页的示例链接。

页脚：一句格言。

最能体现“软件作品”的特点
虽然这是一个简单的单页面，但以下几点体现了它不仅仅是“一个页面”，而是具备“软件工程”思维的软件作品：

模块化结构：HTML 使用语义化标签（如 <div class="card">, <div class="bio">），CSS 采用组件化的命名方式（如 .tag.interest），便于后续维护和扩展。

版本管理：通过 Git 进行代码版本控制，提交历史清晰，体现了协作和迭代开发的潜力。

环境分离：区分了开发环境（本地代码）和生产环境（Gitee Pages部署），这是实际项目部署的标准流程。

用户体验设计：并非简单罗列信息，而是考虑了视觉层次（头像最大最醒目）、交互反馈（卡片和标签的悬停效果），体现了以用户为中心的设计思想。

🤖 AI辅助开发说明
是否使用AI：是

使用的工具：ChatGPT (GPT-4 模型)

关键提示词：

“帮我写一个个人数字名片页面的HTML结构，需要包含姓名、学号、专业、一句话签名、至少三个兴趣标签和三个技能标签、100字自我介绍、头像占位、一个外部链接。”
“为这个HTML结构设计一套现代、简洁的CSS，要求使用毛玻璃效果、柔和的圆角、舒适的间距，标签要有两种不同的颜色区分‘兴趣’和‘技能’。”
“生成一个Gitee仓库的README.md模板，要求包含作品名称、简介、内容说明、文件结构、公开链接，并且要包含AI辅助开发声明部分。”
AI主要帮助了哪些部分：

生成了初始的HTML结构模板，节省了手动编写骨架的时间。

提供了CSS的灵感，特别是毛玻璃效果(backdrop-filter)的实现和颜色搭配方案。

优化了README的书写格式，使其更符合技术文档的规范。

我对AI结果做了哪些修改：

内容个性化：将AI生成的占位文本（如“张三”）替换为我的真实/昵称信息，并将自我介绍修改为更贴合我个人经历的文字。

样式微调：调整了卡片的内边距、行高和一些颜色值，使其阅读体验更舒适。修改了头像的边框样式。

结构优化：在HTML中添加了更细致的class命名，方便自己控制样式，并修正了图片路径。

链接替换：将示例链接替换为指向实际Gitee主页的链接。

我认为AI最不可靠的特点：

生成内容的准确性：AI有时会生成不存在的属性或过时的HTML标签，需要开发者具备基础知识去甄别。

缺乏真实语境：AI无法真正理解“我的”自我介绍应该是什么，它生成的100字往往是套话，必须由我本人重写才能体现个性。

细节把控不足：AI给出的CSS可能在宏观上好看，但在具体的像素级对齐、移动端适配（响应式）细节上常常需要人工大幅度调整。
